# Smoke-чеклист (ручной, перед деплоем)

Компактный ручной прогон ключевых потоков перед деплоем. Покрывает то, что не
ловят unit-тесты: реальный client⇄server socket-флоу, UI, таймеры. Особенно важен
после правок socket-хендлеров (валидация, менеджеры) — см. `docs/review.html` §5.

## Запуск дев-окружения

```bash
# 1) БД (один раз; нужно для auth/статистики — иначе гостевой режим)
docker compose up -d postgres

# 2) Миграции (применит уже закоммиченную 20260617000000_init)
bun run --filter @boardgames/db migrate:deploy

# 3) Сервер (порт 3001; без DATABASE_URL/JWT_SECRET — гостевой режим)
DATABASE_URL=postgres://boardgames:boardgames@localhost:5432/boardgames \
JWT_SECRET=dev-secret-change-me \
bun run dev:server

# 4) Фронт (порт 5173, в отдельном терминале)
cd apps/web && bun install && bun run dev
```

Открыть `http://localhost:5173`. Прогнать по одной игре. Чекбоксы — отметить
вручную; если пункт упал — записать шаги и сообщение из `game:error`/консоли.

## Поток 1 — Codenames (команды, бот-капитан, ход)

- [ ] Создать комнату с хоста → получили код, открылся стол
- [ ] Войти вторым игроком по коду (другая вкладка/инкогнито)
- [ ] Сесть мастером красных (хост); синим поставить **бота** (`room:setCaptain`)
- [ ] В обоих командах есть отгадывающий (хост — красные, гость — синие)
- [ ] Хост меняет настройки в лобби (`room:settings`): риск бота, таймер —
      применяется, бродкаст виден второй вкладке
- [ ] Нажать «Начать» → поле 5×5, капитан видит ключ (цвета под карточками),
      отгадывающий — не видит (owner скрыт)
- [ ] Бот-синий даёт подсказку автоматически (если ход синих); для красных —
      хост вводит подсказку (`game:clue`), число корректное
- [ ] Отгадывающий открывает карточку (`game:guess`) → открывается, лог хода
      обновляется; верное слово — ход остался у команды (видно таймер-бонус)
- [ ] Чужое/нейтральное → ход перешёл; убийца → мгновенный финал
- [ ] Чат (`chat:send`): сообщение появляется у обоих, длинное обрезается
- [ ] Пас (`game:pass`) → ход переходит
- [ ] Партия доходит до финиша → экран результата, rematch (`room:newRound`)
- [ ] Реконнект: перезагрузить вкладку отгадывающего → состояние восстанавливается
      (`room:rejoin` по сессии из localStorage)

## Поток 2 — Uno (рука, ходы, спецкарты, UNO!)

- [ ] Создать комнату, добавить 1-2 ботов, старт → раздача по 7 карт
- [ ] Сыграть карту по цвету/числу → анимация, очередь дальше
- [ ] Дикие карты: `wild` → фаза выбора цвета; `wild4` → выбор цвета + жертва
      добирает 4 (проверить и челлендж, если `challengeDraw4` включён в правилах)
- [ ] `+2` → жертва добирает; стэк `+2` если `stackDraw2` в правилах
- [ ] Skip/reverse работают корректно (reverse при 2 игроках = ход снова)
- [ ] Остаться с 1 картой, **не** нажать UNO! → другой игрок ловит (`catch`),
      берётся штраф; нажать UNO! вовремя — не ловится
- [ ] Выигрышная карта → экран результата, очки чужих рук посчитаны
- [ ] `room:settings` в лобби: тогглы правил (jump-in, 7-0, drawToMatch) —
      применяются, видны второй вкладке
- [ ] Реконнект после reload — рука восстанавливается

## Поток 3 — Alias (команды, ведущий, таймер раунда)

- [ ] Создать комнату, 2 игрока (red/blue), старт → первый раунд
- [ ] Ведущий видит слово, угадывающие — нет (или видят соперники, если
      `showOpponents` включён)
- [ ] «Угадано» → +1, следующее слово; «Пропуск» → −1, следующее слово
- [ ] Таймер раунда (30/60/90) тикает; по истечении последнее слово сгорает
- [ ] Смена ведущего по кругу после раунда
- [ ] Достижение `targetScore` в конце полного круга → победитель; равенство →
      доп. круг
- [ ] `room:settings` в лобби: difficulty, roundDuration, targetScore —
      применяются (difficulty влияет на словарь — проверить, что слова не
      повторяются между раундами)
- [ ] Реконнект после reload — состояние раунда восстанавливается

## Поток 4 — Имаджинариум (ассоциации, 3D-стол, Dixit-подсчёт)

- [ ] Создать комнату с хоста → код, открылось лобби (без команд — общий список)
- [ ] Войти 2-мя игроками по коду (другие вкладки) → 3 игрока минимум; до 6
- [ ] Хост меняет настройки: associationSec/choosingSec/votingSec (30/60/90/120),
      handSize (4–8), targetScore (по колоде / 30 / 50 / 70 / 100) → видны другим
- [ ] «Начать» → раздача по handSize карт, 3D-стол грузится (R3F lazy-чанк),
      ведущий — игрок 0, фаза association
- [ ] Ведущий: кликает карту в 3D-руке → выбирается, вводит ассоциацию, «Задать»
      → карта уходит, фаза choosing, таймер выбора
- [ ] Остальные: кликают карту в 3D-руке → «Сдать» → карта уходит рубашкой на стол
- [ ] Когда все сдали → авто-reveal, фаза voting, стол показывает N карт лицом с
      номерами (tableCards видны, owner скрыт)
- [ ] Не-ведущие кликают 3D-карту-слот → «Голосовать» (нельзя за свою — сервер
      вернёт game:error, карта не засчитана)
- [ ] Когда все проголосовали → авто-tally, фаза scoring: стол подсвечивает карту
      ведущего, показывает кто за кого голосовал, дельты очков
- [ ] «Продолжить» → добор по 1 карте, ведущий по кругу, новый раунд association
- [ ] Игра до исчерпания колоды (deck < players при доборе) → финал, экран
      результата, победитель(и) по max счёта, «Новая партия» (хост)
- [ ] Чат (`chat:send`) работает; таймер фаз тикает, по истечении — авто-скип/
      авто-tally сервером (onTimeout без хендлера — проверить broadcast)
- [ ] Реконнект после reload — состояние восстанавливается (`room:rejoin`),
      таймер фазы перевооружается
- [ ] 3D: карта переворачивается (flip), hover-подъём, выбор подсвечен, мобильный
      touch работает (touch-action: none)

## Регрессия валидации (после правок §5)

Эти кейсы проверяют, что zod-периметр отбрасывает malformed-нагрузку, не роняя
сервер. Быстрый способ — вкладка DevTools Console, emit напрямую через сокет:

```js
//拿到 активный сокет нужного неймспейса (проще через React DevTools:
// find нужный hook, достать socket). Например для codenames:
s.emit('chat:send', 12345); // число вместо строки → game:error, не падает
s.emit('game:guess', -1); // отрицательный индекс → game:error
s.emit('game:guess', 'abc'); // строка → game:error
s.emit('room:settings', { game: 'uno', botCaptains: { red: 1, blue: false }, botRisk: 'x' });
// чужой game / не-boolean → game:error
```

Ожидание: сервер шлёт `game:error` с понятным сообщением, **не** роняет комнату,
валидные действия после этого работают. Если сервер упал / комната зависла —
регрессия валидации (схема пропускает или ломает guard).

## Auth (если включён, с DATABASE_URL)

- [ ] Регистрация → токен, профиль; повторная регистрация того же email → 409
- [ ] Логин → токен; неверный пароль → 401
- [ ] Аватар: загрузка data-URL → сохраняется; слишком большой → 413;
      не-data:image/ → 400
- [ ] `/profile`: статистика по всем 4 играм, история, лидерборд
- [ ] Никнейм из профиля подставляется при входе в комнату

## Что НЕ покрывает этот чеклист

- Масштаб/нагрузка (single-node; Redis-адаптер — открытая MED §10)
- Persist комнат после рестарта сервера (комнаты в RAM — открытая MED §1)
- e2e Playwright (нет авто-тестов — открытая MED §8; этот чеклист — ручной
  заменитель до их появления)
