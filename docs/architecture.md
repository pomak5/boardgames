# Архитектура

Платформа настольных игр онлайн. Сервер авторитарен, игровая логика — чистый TS в
`packages/shared`, реалтайм на socket.io. Self-hosted (VPS), без Convex.

## Стек (актуальный)

| Слой              | Технология                          | Почему                                   |
| ----------------- | ----------------------------------- | ---------------------------------------- |
| Язык              | TypeScript везде                    | Общие типы фронт/бек, меньше рассинхрона |
| Рантайм / PM      | bun + bun workspaces                | Быстро, один тулчейн, нативный TS/тесты  |
| Фронтенд          | React 19 + Vite 7                   | Быстрая сборка, экосистема               |
| Роутинг/анимации  | react-router-dom, framer-motion     | Экраны игр, раздача/броски карт          |
| Стили             | Tailwind CSS + CSS-переменные       | Токены дизайна из `design/final.html`    |
| Сервер            | Fastify + socket.io                 | Лёгкий HTTP (health/auth) + реалтайм     |
| Реалтайм          | socket.io, неймспейс на игру        | Комнаты, reconnect из коробки            |
| Аутентификация    | JWT (jose) + пароли (Bun.password)  | Опциональные аккаунты; иначе гость       |
| БД                | PostgreSQL + Prisma (`packages/db`) | Аккаунты, статистика, словари Alias      |
| Состояние комнат  | В памяти процесса                   | Не усложняем; Redis — если масштабировать |
| Деплой            | Docker Compose на VPS; фронт статикой | Просто поднять на любом сервере         |

## Структура

```
apps/server/src
  index.ts            # bootstrap: Fastify (health) + socket.io; крутит реестр games
  games.ts            # GameModule[] — реестр игр (точка расширения)
  auth/               # jwt.ts, password.ts, routes.ts (маунт только при DATABASE_URL)
  codenames/          # manager.ts (логика комнат) + handlers.ts (socket-хендлеры)
  uno/                # manager.ts + handlers.ts

apps/web/src          # STANDALONE Vite-проект (вне workspaces, biome)
  play/
    home/             # хаб выбора игры
    net/socket.ts     # клиент socket.io
    online/           # общий онлайн-флоу: лобби, чат, комната
    codenames/        # экраны + useCodenamesGame
    uno/              # экраны + useUnoRoom
  components/ui/      # только используемые shadcn-компоненты

packages/shared/src   # ЕДИНЫЙ движок (чистый TS, без I/O) + тесты
  index.ts            # barrel
  events.ts           # типизированные socket-события client⇄server
  roomCode.ts
  codenames/          # types, engine, dictionary, bot, coop, view
  uno/                # types, deck, rules, round, play, special, draw, timeout, engine, bot, view

packages/db           # Prisma client + schema (User, GameResult, AliasWord)
```

Фронт не дублирует движок: импортит `packages/shared` через алиас `@shared`
(Vite + tsconfig paths). Единый источник правды для правил и типов.

## Ключевые принципы

1. **Игровая логика — чистые функции в `packages/shared`**, без I/O:
   `applyAction(state, action) -> newState | error`. Отсюда: сервер — источник
   правды (анти-чит), лёгкие юнит-тесты правил, боты на том же движке.
2. **Сервер авторитарен.** Клиент шлёт намерения (`playCard`), сервер валидирует
   и рассылает новое состояние. Каждый игрок получает только видимую ему часть
   (чужие карты закрыты — `view`/redacted).
3. **Реестр игр.** Каждая игра — `GameModule` в `apps/server/src/games.ts`;
   `index.ts` поднимает socket.io-неймспейс на каждую запись. Новая игра = модуль
   движка + серверный модуль + одна строка в реестре (см. CLAUDE.md).
4. **Типизированные события.** Все socket-события — в `packages/shared` (`events.ts`
   и `uno/types`), фронт и бек не разъезжаются.
5. **Reconnect-friendly.** Состояние комнаты живёт на сервере; при переподключении
   клиент получает снапшот. Гостевая сессия / аккаунт — JWT.

## Решения по продукту

| Вопрос        | Решение                                                            |
| ------------- | ----------------------------------------------------------------- |
| Как играть    | Комнаты по коду/ссылке (подбор случайных игроков — в планах)       |
| Аккаунты      | Можно играть **гостем по нику**; аккаунты опциональны (JWT)        |
| Мобильные     | Mobile-first вёрстка                                               |
| Языки         | Русский (строки в словарь — i18n-ready)                            |
| Общение       | Текстовый чат в комнате                                            |
| Боты          | Включаются в настройках комнаты; число игроков настраивается       |
| Uno           | Классика по умолчанию; 7 вариаций правил — тогглами в лобби        |
| UNO!          | Кнопка «UNO!» со штрафом за пропуск + поимка                       |
| Alias (план)  | Команды, таймер, очки со штрафом; ведущий подтверждает слова       |
| Словари Alias | Наборы по сложности (таблица `AliasWord`)                          |
| Дизайн        | Утверждён `design/final.html` («Уютный вечер» / «Лаунж»)           |

## База данных

Схема — `packages/db/prisma/schema.prisma`:

- `User` — опциональные аккаунты (логин + хеш пароля).
- `GameResult` — история партий (для статистики).
- `AliasWord` — слова: `word`, `difficulty`.

Активные комнаты и состояния игр в БД **не** храним — только память процесса.

## Жизненный цикл комнаты

`lobby → playing → finished → (новый раунд | закрытие)`. Пустые комнаты убираются
по TTL.
