| Мобильные | Mobile-first вёрстка с первого дня |

## Стек

| Слой | Технология | Почему |
| --- | --- | --- |
| Язык | TypeScript везде | Общие типы фронт/бек, меньше рассинхрона |
| Фронтенд | React 18 + Vite | Быстрая сборка, экосистема |
| Состояние | Zustand | Просто, без бойлерплейта |
| Стили | Tailwind CSS + CSS-переменные (токены из выбранного дизайна) | Скорость + системность |
| Анимации карт | Framer Motion | Раздача, броски, веер в руке |
| Бекенд | Node + Fastify | Лёгкий, быстрый |
| Реалтайм | Socket.IO | Комнаты, reconnect из коробки |
| БД | PostgreSQL + Prisma | Аккаунты, статистика, словари Alias |
| Кеш/состояние игр | В памяти процесса (MVP) → Redis (потом, при масштабировании) | Не усложняем раньше времени |
| Монорепо | npm workspaces + turborepo (опц.) | Один клон, общие пакеты |
| Деплой | Docker Compose; фронт — статика за Caddy/nginx | Просто поднять на любом VPS |

## Структура

```
apps/web                 # SPA
  src/
    app/                 # роутинг, layout
    features/
      lobby/             # главная, создание/поиск комнат, подбор
      room/              # комната: игроки, чат, настройки
      games/uno/         # стол, рука, анимации
      games/alias/       # раунд, таймер, слова, счёт
      auth/              # гость / аккаунт
      chat/
    shared/ui/           # дизайн-система (токены из выбранного концепта)

apps/server
  src/
    modules/
      auth/              # гостевые сессии (JWT) + аккаунты
      rooms/             # создание, код-ссылка, настройки, lifecycle
      matchmaking/       # очередь подбора случайных игроков
      chat/
      games/             # оркестратор: подключает движки из shared
    infra/               # prisma, socket.io, config

packages/shared
  src/
    types/               # Player, Room, GameState, ...
    events/              # типизированные socket-события (client⇄server)
    games/uno/           # ЧИСТАЯ логика Uno: колода, ходы, валидация
    games/alias/         # ЧИСТАЯ логика Alias: раунды, очки
```

## Ключевые принципы

1. **Игровая логика — чистые функции в `packages/shared`**, без I/O.
   `applyAction(state, action) -> newState | error`. Это даёт:
   - сервер — единственный источник правды (анти-чит);
   - лёгкое юнит-тестирование правил;
   - боты используют тот же движок.
2. **Сервер авторитарен.** Клиент шлёт намерения (`playCard`), сервер валидирует и рассылает новое состояние. Каждый игрок получает только видимую ему часть состояния (чужие карты — закрыты).
3. **Reconnect-friendly.** Состояние комнаты живёт на сервере; при переподключении клиент получает снапшот. Гостевая сессия — JWT в localStorage.
4. **Типизированные события.** Все socket-события описаны в `packages/shared/events` — фронт и бек не могут разъехаться.

## Потоки

### Комната по ссылке
создать комнату → код `ABCD12` + ссылка → друзья заходят (гость/аккаунт) → хозяин настраивает (игра, кол-во игроков, боты) → старт.

### Подбор случайных игроков
«Играть» → очередь по игре → набралось достаточно (или таймаут → доукомплектация ботами, если включено) → авто-комната → старт.

### Жизненный цикл комнаты
`lobby → playing → finished → (rematch | закрытие)`. Пустые комнаты удаляются по TTL.

## База данных (MVP)

- `users` — опциональные аккаунты (email/пароль или OAuth позже)
- `guest_sessions` — ник + JWT id (для reconnect)
- `alias_words` — слова: `word, difficulty (easy|medium|hard)`
- `game_results` — история партий (для будущей статистики)

Активные комнаты и состояния игр в БД **не** храним (память процесса).
